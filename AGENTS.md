# Emoji math — Agent Reference

## Overview

**Emoji math** is an iOS math quiz game built with **UIKit** and **storyboards**. Players solve arithmetic equations where operands are displayed as themed emoji images (hats, animals, fruits, etc.) instead of digits. The last equation's result is hidden and the player picks the correct answer from four multiple-choice buttons under a countdown timer.

- **Bundle ID:** `nirhayun.Emojimath`
- **Display name:** Emoji math
- **Version:** 1.1.0 (build 1)
- **Min iOS (Podfile):** 15.0
- **Language:** Swift (project file says 4.0 but source uses modern patterns like `@main`)

> **Historical note:** the project was originally named "Math Quiz Objects". Some test bundle IDs and pbxproj references still use that name.

## Architecture

### Navigation Flow

```
SplashScreen (4s tip) → StartScreen (main menu) → MainController (gameplay)
                                                  → PassStage (level-up overlay)
                                                  → CustomAd (fallback promo)
```

All navigation is **storyboard-driven** with `Main.storyboard` as the entry point. `SplashScreen` is the initial VC; it presents `StartScreen` after a delay.

### State Management

- **`SaveData`** — thin `UserDefaults` wrapper for all persistent state: score, best score, three hint counts (capped at 99), sound preference, daily login/streak dates, reward-ad timestamps, Game Center flags.
- **`AppDelegate.timer`** — shared `Int` used by `MainController` for the countdown; adjusted on background/foreground transitions.
- No Core Data, no Realm, no external database.

### Monolithic View Controllers

`StartScreen` (~850 lines) and `MainController` (very large) contain most of the app logic. Newer singleton helpers (`AdManager`, `AudioManager`, `GameCenterManager`, `Constants`) represent a **partial refactor in progress** — they are not yet fully adopted.

**Build:** Always open `Emoji math.xcworkspace` (not `.xcodeproj`) so CocoaPods are linked.

## Ad Integration

There are **two parallel ad implementations** — this is an active inconsistency:

| Location | API Style | Used By |
|----------|-----------|---------|
| `StartScreen.swift` | Legacy `GADInterstitial` / `GADRewardBasedVideoAd` with hard-coded unit IDs | Main menu reward flow |
| `AdManager.swift` | Modern `GADInterstitialAd` / `GADRewardedAd` singleton | `PassStage` (level-up) |

`Constants.swift` defines ad unit IDs with `#if DEBUG` / `#else` toggling between Google test IDs and production IDs. `StartScreen` still uses inline string literals for its ad IDs.

When an AdMob interstitial fails to load, `CustomAd` is shown as a local promo fallback (links to Facebook/Instagram).

## Game Logic

### EquationGenerator

- Builds a set of intermediate equations plus one **final** equation whose answer becomes the quiz question.
- Uses **`NSExpression`** to evaluate string math expressions.
- Generates three wrong alternative answers for multiple-choice buttons.
- Known typo: tuple label `umberOfVariables` (missing leading "n") propagated to `Stages`.

### Stages

`Stages.stages` is an array of tuples defining each level: variable count, number range, timer seconds, allowed operations, difficulty enum, theme name, emoji icon set, and UI colors. Score thresholds go up to 7000+.

### Hints

Three hint types, each with a persistent count in `SaveData`:
1. **Reveal** — shows a hidden number
2. **50/50** — disables two wrong answer buttons
3. **Extra time** — adds seconds to the countdown

---

## Game Center

Leaderboard IDs (defined in `Constants.GameCenter`):
- `BSTP` — Best Score to Pass
- `LWST` — Lowest Score
- `RCBP` — Record Best Pass
- `NMPC` — Number of Passes Completed

`StartScreen` uses the older `GKScore.report` API. `GameCenterManager` wraps the newer `GKLeaderboard.submitScore` but is **not yet wired into** `StartScreen`.

---

## Notifications

`StartScreen.setDailyNotifications()` schedules weekly repeating local notifications via `UNUserNotificationCenter`. `Constants.Notifications.daily` has structured copy for each weekday but is not yet used by the inline scheduling code.

## Key Known Issues & Inconsistencies

1. **iOS deployment target mismatch** — `Podfile` says 15.0; `project.pbxproj` still has 10.3 on app/test targets.
2. **Swift version** — `project.pbxproj` declares `SWIFT_VERSION = 4.0` but source uses modern Swift features.
3. **Dual ad systems** — Legacy ad API in `StartScreen` vs modern API in `AdManager`; should be consolidated.
4. **Unused singletons** — `AudioManager` and `GameCenterManager` are defined but not adopted by the main VCs (they use local `AVAudioPlayer` instances and `GKScore` directly).
5. **Facebook SDK bridging header** — `Emoji math-Bridging-Header.h` imports FBSDK but no FB pod is in the Podfile and all FB Swift code is commented out. The header is legacy.
6. **Sensitive file** — `GoogleService-Info.plist` contains real Firebase/API keys; `.gitignore` lists it but verify it is not tracked.

## Conventions for Agents

- **Always open the `.xcworkspace`**, never the `.xcodeproj`.
- **Do not commit** `GoogleService-Info.plist` or anything under `Pods/`.
- When modifying ads, prefer the `AdManager` singleton pattern over inline legacy code.
- `SaveData` is the single source of truth for persistent state — do not access `UserDefaults` directly elsewhere.
- Stage/level data lives in `Stages.swift`; equation logic in `EquationGenerator.swift` — keep them separate.
- UI is storyboard-based; `@IBOutlet` and `@IBAction` bindings are in `Main.storyboard`.
- Sound files are `.mp3` in the app bundle root; `AudioManager` has a preload list but `MainController` still creates players inline.
- The partial refactor (`Constants`, `AdManager`, `AudioManager`, `GameCenterManager`, extensions) should be completed over time — new code should use these modules rather than adding more inline logic to the VCs.
